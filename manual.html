Chol DictMaker Application Manual

Chol DictMaker is a desktop tool designed for creating, managing, and editing multilingual dictionary projects using an SQLite database backend.

1. Project Management (File Menu)

The File menu is the starting point for all dictionary work. Use shortcuts where available for speed.

  • New Project (Ctrl+Shift+N): Opens the New Dictionary Project Setup dialog. You must specify a Database File Name (e.g., `my_language.db`) and Define Dictionary Fields (Columns). The `headword` field is mandatory for indexing/search, and the `meaning` field is added automatically as a repeatable field.
  • Open Project (Ctrl+O): Prompts the user to select an existing SQLite database (`.db`) file. The application reads the table schema to configure the entry form.
  • Recent Projects: A submenu displaying the last 5 projects for quick access.
  • Import Data: Allows importing entries from CSV or JSON files. The import expects fixed fields (like `headword`, `pos`) and the repeatable field named `meaning`.
  • Export Data: Allows exporting all dictionary entries to CSV or JSON formats.
  • Exit (Ctrl+Q): Closes the application after confirming unsaved data status.

2. Data Entry and Editing (Entry Form Tab)

This is the primary interface for creating and modifying dictionary entries.

Entry Form Navigation and Controls

  • New Entry: Resets the form to clear all fields and prepare for a new entry.
  • Save / Update Entry (Ctrl+S): Displays Save New Entry (performs `INSERT`) or Update Entry (ID: {ID}) (performs `UPDATE`). Validation: The `Headword` field cannot be empty.
  • Delete Entry (Ctrl+D): Only visible when editing an entry; permanently deletes the current entry after confirmation.

Field Types

  • Fixed Fields (Headword, POS, Custom): Rendered as single-line (`QLineEdit`) or multi-line (`QTextEdit`) input. The `pos` (part-of-speech) field includes an auto-completer feature based on existing terms.
  • Repeatable Fields (e.g., Meaning): Displayed in a vertical section. Use the Add (+) button to add new input fields (`RepeatableFieldWidget`), allowing one entry to have multiple associated meanings/examples.

3. Viewing and Searching

3.1 Headword Index Panel (Left Panel)

Displays a paginated list of all dictionary Headwords in alphabetical order.

  • Pagination: Use ◀ and ▶ buttons to navigate. The current status (e.g., "Page 1/X") is displayed.
  • Loading: Clicking any item loads that entry immediately into the Entry Form tab for editing.

3.2 Search Tab (Viewer)

Allows comprehensive searching across the database using various methods:

  • Search Input: Filters entries across all fixed fields and the `meaning` texts.
  • Filter Input: Allows filtering specifically by the `pos` field (e.g., "Noun").
  • Alphabet Filter Buttons: Dynamic buttons generated from unique first letters of headwords. Clicking All clears the filter.
  • Search Results List: Displays results with a formatted summary. Double-clicking a result loads it into the Entry Form.

4. Settings and Tools

4.1 Project Menu

  • Project Details (Ctrl+I): Displays key statistics (File location, size, headword count, last edited time).
  • Headword Duplicates / Meaning Duplicates: Runs queries to find and report duplicate entries (using a normalized check).

4.2 View Menu

  • Toggle Headword Panel (Ctrl+P): Shows or hides the left-side index panel.
  • Zoom In (Ctrl++) / Zoom Out (Ctrl+-): Adjusts global UI scale (50% to 200%).
  • Change Theme: Selects between available themes (e.g., Light, Dark).

4.3 Settings Menu

  • Font Settings: Opens a dialog to customize the font and size for the Headword Field and Repeatable Fields (saved per project).
  • Change Language: Allows selecting the Application's UI language. A restart is required to apply the change fully.

4.4 Tools Menu

  • Export to PDF: Opens the PDF Export dialog to save all dictionary entries to a formatted PDF document.
  • Database Management: Opens the Database Tools Dialog for Backup Database and Restore Database operations.
